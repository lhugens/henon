Introduction

Given a specific dynamical system, with an associated map r n+1 =F( r n ) , it is possible to define an escape time t( r ) as the number of the map iterations necessary for the trajectory to leave a certain domain Γ . The main goal of this study is to perform a comparative analysis between computer algorithms specificaly designed for an efficient search of long trajectories and generic maximization algorithms.

Map Parameters

For this study, we used the iconic Hénon Map with k=6 :
x n+1 =k- x n ²- y n
y n+1 = x n
The associated Jacobian matrix is J( x ) =( -2 x n -1 1 0 ) , and the respective determinant is J( x ) =1 . Its eigenvalues and eigenvectors are respectively e ± =-x ± x²-1 and e ± =( e ± ,1 ) .
The Lyapunov exponent is, therefore:
λ n ( x ) = 1 n ∑ i=0 t-1 log( J( x ) ) =0
The largest escape time generated yet by a multicanonical simulation on Γ =[ -5,9 ] ² algorithm is 627 map iterations.

Generic Maximization Algorithms

The source https://docs.scipy.org/doc/scipy/reference/tutorial/optimize.html provides several generic minimization algorithms implemented in the scipy.optimize package, ready to use.
The following algorithms are automatically unusable due to the need of a gradient of the function to be minimized, which is generaly impossible to obtain due to the roughness of the landscape: Newton-Conjugate-Gradient (NCG), Trust-Region NCG, Trust-Region Truncated Generalized Lanczos / NCG, Trust-Region Nearly Exact.
At first sight, the usable algorithms are those who only rely on the evaluation of the function, those which belong to the Pattern Search family (also known as direct search, derivative search, or black-box search).

0.3 Powell's Method

References: Brent, Richard P. (1973). "Section 7.3: Powell's algorithm". Algorithms for minimization without derivatives; Flannery, BP (2007); Numerical Recipes: The Art of Scientific Computing.
To understand this method, we must first have the definition of directions. Consider the following Taylor series approximation to a function f( X ) and a point P :
f( x ) =f( P ) + ∑ i ∂ f ∂ x i x i + 1 2 ∑ i,j ∂ 2 f ∂ x i ∂ x j x i x j + ⋯ ≈ c- b ⋅ x + 1 2 x ⋅ A ⋅ x
where
c ≡ f( P ) b ≡ - ∇ f | P [ A ] ij ≡ ∂ 2 f ∂ x i ∂ x j | P
and the approximation to the functions gradient:
∇ f= A ⋅ x - b
Therefore, the gradient will vanish - the function will be at a neighbouring extremum - at P+x where x is the solution to A ⋅ x = b . If we move along that direction to a minimum and now consider moving in a direction v , in order not to ruin the previous minimization, we pretend that the gradient continues to be perpendicular to x . Considering a change in the gradient:
δ ( ∇ f ) = A ⋅ ( δ v )
we therefore are interested in directions v that satisfy:
0= u ⋅ δ ( ∇ f ) = u ⋅ A ⋅ v
When this equation holds for two directions u and v , they are said to be . Powell's method produces n mutually conjugate directions for an objective n -dimensional function, and needs an 1-dimensional minimization algorithm such as Golden Search to minimize f( x ) in a particular direction. 1 The Rosenbrock function

Figure 2: Rosenbrock function ( z=f( x,y ) ) for a=1  and  b=100 .
The Rosenbrock function (represented in the figure above) is a non-convex function which is used as a performance test problem for optimization algorithms. The global minimum is inside a long, narrow, parabolic shaped flat valley, and minimization algorithms usually converge rapidly to this valley, but converging to the actual global minimum is harder. The function is defined as:
f( x,y ) =(a-x ) 2 +b ( y- x 2 ) 2
Its partial derivatives are:
∂ f ∂ x ( x,y ) =2( a-x ) -4bx( y- x 2 )
∂ f ∂ y ( x,y ) =2b( y- x 2 )
Calculating extremum points:
∂ f ∂ y ( x,y ) =0 ⟹ y= x 2
( y= x 2 ⋀ ∂ f ∂ x ( x,y ) =0 ) ⟹ ( x=a ⋀ y= a 2 )
f( a, a 2 ) =0
The global minimum is therefore located at ( a, a 2 ) and its value is 0 .
